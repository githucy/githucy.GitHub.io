hexo 搭建个人博客

操作步骤

  1. 安装node.js
    网上搜索node.js下载地址，下载后安装,使用node -v查看node.js版本,npm -v查看npm版本.
  2. 安装hexo客户端及使用
    安装：执行npm install hexo-cli -g，初始化文件夹：hexo init 文件夹名，进入文件夹：cd 文件夹名，安装依赖：npm install，生成静态页面：hexo g，启动本地服务器：hexo s，访问：http://localhost:4000/，即可预览搭建后的效果。
  3. 在GitHub新建一个仓库，依次执行以下命令：
    echo “# hexo-blog” >> README.md
    git init
    git add README.md (git add . #用于添加全部文件)
    git commit -m “first commit”
    git branch -M main
    git remote add origin https://github.com//hexo-blog.git
    git push -u origin main
    (即GitHub 新建一个仓库，后所给出的代码)
  4. 修改_config.yml配置文件，填上服务器的部署配置
    deploy:
    type: git
    repository： git@github.com//.git（yourrepo为仓库名）
    branch： main
  5. 创建新文章
    hexo new post “文章标题”
    hexo new page “页面标题”
    hexo new draft “草稿标题”
    hexo new source “资源文件名”
    hexo new tag “标签名”
    hexo new category “分类名”
    hexo new photo “相册名”
    hexo new link “友情链接名”
    hexo new about “关于”
    打开创建的文件，修改内容，保存即可。
  6. 利用三合一的命令发布到GitHub
    hexo clean && hexo generate && hexo deploy或hexo clean; hexo generate; hexo deploy

注意事项

  1. 报错代码 #（PS E:\vs\my-blong> git commit -m “first commit” Author identity unknown

*** Please tell me who you are.

Run

git config –global user.email “you@example.com“ git config –global user.name “Your Name”

to set your account’s default identity. Omit –global to set the identity only in this repository.

fatal: unable to auto-detect email address (got ‘asus@LAPTOP-0HRVOJJ6.(none)’））#
错误信息 fatal: unable to auto-detect email address 表明 Git 未配置用户信息。
使用以下命令设置用户名和邮箱：
git config –global user.name “Your Name”
git config –global user.email “your.email@example.com“
验证
git config –global user.emailw
git config –global user.name
如果输出是你设置的邮箱和用户名，说明配置成功。
2. 报错代码
在git add . 时候出现如下错误：
warning: LF will be replaced by CRLF in .gitignore.
这个错误是因为git默认使用windows的换行符LF，而mac和linux系统默认使用CRLF。
解决方法是：
git config –global core.autocrlf false
这个命令会告诉git不要转换换行符，这样在mac和linux系统上，git add . 时不会出现错误。
3. YAMLException 错误
错误信息表明 _posts/试验.md 文件的 YAML Front Matter 部分存在语法问题。具体是 tags 后缺少空格，导致解析失败。
错误代码：
tags:操作步骤及注意事项
正确格式：
tags: 操作步骤及注意事项
4. Deployer not found: git
这个错误表明 Hexo 的 git 部署插件未安装。
解决方法
修复 _posts/试验.md 文件的 YAML Front Matter
打开 试验.md 文件，将 tags 行修正为以下内容：
tags: 操作步骤及注意事项

完整修正后的 Front Matter 示例：

title: hexo 搭建个人博客
date: 2025-05-08 14:22:12
tags: 操作步骤及注意事项

保存文件后重新运行命令。
安装 Hexo 的 git 部署插件
在终端中运行以下命令安装 hexo-deployer-git 插件：
npm install hexo-deployer-git –save
安装完成后，重新运行以下命令：
hexo clean; hexo generate; hexo deploy
